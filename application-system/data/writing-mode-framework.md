# Writing Mode Framework

This note explains how Nguyen Le Cuong's authentic writing voice should be transformed across four different output modes.

The base rule is:

- keep the same underlying human voice DNA
- change the surface style depending on the document purpose

Do not treat one mode as suitable for all outputs.

## 1. Blog mode

Use for:
- blog posts
- reflective essays
- product stories
- learning-in-public posts
- personal technical writing

Core feel:
- human
- reflective
- exploratory
- systems-oriented
- sometimes playful
- can shift into mentor mode

What to preserve:
- real trigger or reason for writing
- lived context
- tradeoffs and failures
- short takeaway lines when useful
- visible thought process
- personal curiosity

What to suppress:
- stiff academic caution
- overly corporate phrasing
- generic AI-blog hooks

Typical shape:
- story
- friction
- aha moment
- practical lesson
- what I would do differently

## 2. Application mode

Use for:
- cover letters
- recruiter emails
- application summaries
- role-fit explanations

Core feel:
- sincere
- formal
- contribution-oriented
- recruiter-safe
- personally grounded, but controlled

What to preserve:
- real reason for applying
- human motivation
- confidence through contribution
- evidence from lived work

What to suppress:
- blog tangents
- overly emotional reflection
- playful or exploratory sprawl
- mentor/preacher tone
- slogan-like lines unless extremely restrained

Typical shape:
- why this role
- what makes the fit credible
- why this company/context matters
- concise closing

Hard rule:
- the application system should always use `application mode` only

## 3. Work mode

Use for:
- user stories
- ticket descriptions
- meeting recaps
- Slack / Teams updates
- requirement clarification
- decision notes
- stakeholder follow-up messages
- process notes

Core feel:
- clear
- direct
- structured
- calm
- action-oriented
- low-ego

What to preserve:
- systems thinking
- practical reasoning
- honest visibility into blockers, tradeoffs, and next steps
- enough humanity to stay readable

What to suppress:
- blog reflection
- recruiter polish
- mentor/preacher tone
- slogan-like lines
- unnecessary emotional language

Typical shape:
- context
- current state
- decision / ask / blocker
- next step

## 4. Academic mode

Use for:
- research papers
- theses
- literature reviews
- methodology sections
- class submissions

Core feel:
- precise
- citation-driven
- restrained
- logically structured
- human, but not conversational

What to preserve:
- systems thinking
- clear explanation
- problem framing
- tradeoff awareness
- readable logic

What to suppress:
- direct reader talk
- blog energy
- mentor-style punchlines
- emotional emphasis
- casual bilingual rhythm unless explicitly appropriate

Typical shape:
- research question
- gap
- method
- evidence
- interpretation
- limitation
- conclusion

## 5. Shared underlying DNA

Across all four modes, keep:
- sincerity
- practical reasoning
- systems awareness
- clarity about what was learned
- honesty about limits

Across all four modes, avoid:
- inflated claims
- fake authority
- generic AI smoothness
- empty hype

## 6. Transformation rule

The same idea may appear differently in each mode.

Example:

- Blog mode:
  - `Roadmap là một thực thể sống.`
- Application mode:
  - `I work best in environments where plans need to be adapted carefully as priorities change.`
- Work mode:
  - `The roadmap should be treated as a live planning artifact and reviewed when priorities or dependencies change.`
- Academic mode:
  - `Roadmaps should be treated as adaptive planning artifacts rather than fixed commitments, particularly in dynamic delivery environments.`

Future agents should decide the mode first, then draft.
