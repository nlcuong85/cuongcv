## Remaining gaps

These are not blockers for Phase 4, but they are still real limitations.

### 1. Role-family classification quality is usable, not excellent

Current corpus result:

- exact match: 52.5%
- compatible match: 72.5%

Meaning:

- good enough for front-door filtering
- not yet strong enough for fully trusted automatic role-family assignment

### 2. Live scanning is still minimal

Current future-lead intake works through:

- manual add via JSON or CLI arguments

Still missing:

- richer ATS/API scanning directly into raw leads for local daily use

### 3. Contact lookup is still draft-level in the prototype

The promoted draft intake can document fallback status, but it does not yet run a robust recruiter lookup before writing the live intake.

### 4. No analytics or follow-up layer yet

Compared with the stronger `career-ops` operating model, this prototype still lacks:

- pattern analysis
- follow-up cadence
- tracker dashboard

## Conclusion

Phase 4 is functionally met for Cuong's repo in the sense that:

- future jobs now have a front door
- they can be filtered before application generation
- approved leads can enter the real application workflow
- the full path was tested end-to-end

Phase 4 is not "perfect" yet because the classification and live scanning layers still need improvement, but it is now coherent and usable.
