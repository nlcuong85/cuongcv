# Data Contract

This prototype separates stable prototype logic from Cuong-specific search data and generated experimental artifacts.

## System Layer

These files define the prototype itself and are safe to edit as part of prototype development:

- `README.md`
- `DATA_CONTRACT.md`
- `docs/*`
- `scripts/*`

## User / Search Policy Layer

These files define Cuong-specific search behavior for this experimental subsystem:

- `config/profile.yml`
- `config/portals.yml`

These should be treated as the prototype's personalization layer.

## Generated Experimental State

These files are outputs of the prototype:

- `data/current_corpus.json`
- `data/current_corpus_summary.md`
- `data/raw_leads.jsonl`
- `data/franklee_*_queue.json`
- `data/franklee_*_queue.md`
- `data/pipeline.md`
- `data/tracker.tsv`
- `data/verification_report.md`
- `data/promotion-map.tsv`
- `data/promotion-preview.md`
- `promotions/*`

They may be regenerated by scripts in this folder.

## Rule

This prototype may read the current application corpus to validate itself.

This prototype must not become the source of truth for:

- live application generation
- generated cover letters
- generated CV variants
- the current `job-search-cuong` skill behavior

Its role is:

- discover
- classify
- filter
- track
- promote selected leads into the current application workflow later
